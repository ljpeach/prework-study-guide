Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Head elements are largely invisible and primarily serve to provide information about a website to search engines and computers.
  • Meta elements contains other information about websites. It is kept within the head element.
  • Title stores the title of the page. This can be used for search engine optimization, and unlike other elements held within head, is usually displayed as text on tabs.
  • Header elements should be used in order, with h1 being recommended to only be used for the primary title of the page.
  • Body elements are responsible for formatting the content on a webpage.
  • An element is everything within brackets including the brackets themselves, while tags are what is between brackets.

CSS

  • CSS files can be used to format entire pages, while inline CSS can be used to format specific elements.
  • Internal style sheets are written within the head section.
  • External style sheets are written in external files and are linked with the head section using a link element.
  • General format is selector {property:value; property:value}
  • The same formatting can be specified for multiple selectors by listing them separated by commas
  • Classes can be used as selectors to specify special rules for certain elements, but not all of that type.
  • Classes are declared by adding a period before the selector in the style sheet. ex: .example {property:value;}
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git is used for version control and storing projects.
  • git commands always begin with git.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: stages all changes.
  • git commit -m "description here": saves staged changes to the branch using the written description to tell developers what was changed. The description should be present tense and as imperative sentances.
  • git push origin branch-name: pushes new commits to the specified branch.
